Review: Modern React with Redux

This is a review of Modern React with Redux offered on Udemy by Stephen Grider. I walked into this knowing very little about webpack, babel, react, redux or ES6. JSX. The first third is pure react, in which you build a functional youtube browser. The last two thirds is Redux, in which you build a booklist app, 5-day weather sparklines app and a functional comments board.

Take-home Messages

  • The emphasis in this class is on doing things from scratch the wrong way, to really understand the parts and how to debug.
  • It includes a hands-on walkthrough of how to break up a UI into components, which puts react in the overall context of a project workflow.
  • During the class I used the IntelliJ editor with the JSX Harmony plugin. This made dealing with JSX really easy.
  • Key principles:
    • “Controlled Component” is bound from state.
    • “Downward Data Flow”
  • React is really simple and easy to use, nearly as simple as knockout.
    • React is much simpler than backbone in terms of number of concepts.
    • Greatly helped by a few tricks that are possible because of ES6.
    • React is most complex than knockout, but the increased flexibility seems worth it.
    • The main downside is that you have to write in ES6, which means for the time being that you need to have a build step, an debugging in the browser is harder.
    • In a simple example the course manages to show why react (and knockout) are not enough for a complex app.
  • Redux is... quite surprising, but simple when you understand it.
    • It is a way of maintaining global state that is composed of immutable data that can only be updated through partial replacements of sub-parts through events.
    • I have never heard a decent description of redux
    • Goes into details of core mechanics of redux.
    • Instructor claims that properly teaching redux is one of the most challenging things he has tried to do.
    • He does a pretty good job, and I came away feeling pretty good about redux
    • Not only is redux more complicated, it is quickly evolving, and often has incomplete or outdated docs
  • Sometimes painfully slow description of events, state, constructors and functional programming, which I pretty much got on the first explanation, but which he went through at length.
  • I ended up watching at least half the videos at 2x speed.

  • redux container/smart component
  • webpack
  • redux reducer
  • redux state vs react state
  • redux middleware, redux promise
  • React router, browserHistory vs hashHistory
